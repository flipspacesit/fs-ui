## Recommended sequencing

1. **Refactor the 10 existing components** to spec (immediate value тАФ current app screens improve on upgrade; non-breaking). Order: Button тЖТ inputs (Text/Select/Date/Search) тЖТ Dropdown/Accordion тЖТ Tag тЖТ Table тЖТ Dialog/ModalLayout тЖТ Loader тЖТ SplitMenu.
2. **Build core missing primitives** most apps need: Checkbox, RadioButton, Switch, Tabs, Tooltip, Badge, Breadcrumb, IconButton, Snackbar/Alert.
3. **Build date & data components**: Calendar/DatePicker family, Card/MediaCard, Stepper.
4. **Domain-specific (Vizdom)**: Pins, SpaceCard, SidebarNav, composites.
5. **Asset sets** (icons/illustrations) as needed by consumers.

Each component: extract its full spec (already captured), implement against **tokens only**, add to the docs site, verify in the preview, keep the public API non-breaking.
